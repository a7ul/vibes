CONCEPT-13 CRITICAL - corrected Graph API (from lib/graph/):
- CORRECT constructor: new Graph<State, Output>([node1, node2], { maxIterations?: number })
  WRONG constructor (old docs): new Graph({ nodes: [node1, node2] })

  WRONG transition (old docs): return this.next("node-id", newState); OR return this.output(value);
- CORRECT Mermaid format: flowchart TD (what toMermaid() generates)
  WRONG Mermaid format (old docs): stateDiagram-v2
- BaseNode<State, Output> - abstract class with abstract id: string, abstract run(state): Promise<NodeResult>
- nextNodes?: string[] - optional array declaring possible next node IDs (used by toMermaid() for edges)
- Graph.run(initialState, startNodeId, options?) - runs to completion
- Graph.runIter(initialState, startNodeId) - returns GraphRun for step-by-step
- GraphRun.next() - returns GraphStep | null; step.kind === "node" (has nodeId, state) or "output" (has output)
- graph.toMermaid() - returns flowchart TD string
- FileStatePersistence<State>(dirPath) - persists state to files; pass as { persistence, graphId } in run options
- MemoryStatePersistence<State>() - in-memory persistence
