CONCEPT-09 key APIs (from lib/execution/deferred.ts, lib/types/errors.ts, lib/agent.ts):
- requiresApproval: true — option on tool() that causes ApprovalRequiredError before execute
- ApprovalRequiredError — caught with instanceof; has .deferred property
- deferred.requests — DeferredToolRequest[]: { toolCallId, toolName, args }
- agent.resume(deferred, results) — resumes run; results shape: { results: [{ toolCallId, result? OR argsOverride? }] }
- ExternalToolset — for tools with JSON Schema (not Zod); auto-sets requiresApproval; takes ExternalToolDefinition[]: { name, description, jsonSchema }

CONCEPT-10 key APIs (from lib/testing/test_model.ts, function_model.ts, mod.ts):
- setAllowModelRequests(false) — module-level guard; prevents accidental real API calls
- new TestModel(options?) — auto-generates tool calls then final_result; options: { callTools?, text?, outputSchema? }
- createTestModel({ outputSchema }) — convenience factory with typed output
- agent.override({ model }) — returns { run, stream, runStreamEvents }; bypasses setAllowModelRequests guard
- new FunctionModel(fn) — fn receives { messages, tools, turn } returns ModelResponse object
- captureRunMessages(async fn) — returns { result, messages: ModelMessage[][] }; sequential only (not concurrent-safe)

CONCEPT-11 key APIs (from lib/otel/instrumentation.ts, otel_types.ts):
- instrumentAgent(agent, options) — wraps agent; options: { functionId, metadata?, excludeContent?, isEnabled? }
- returns same interface as agent: { run, stream, runStreamEvents }
- Agent-level: new Agent({ model, telemetry: TelemetrySettings }) — direct inline telemetry
- TelemetrySettings from "ai": { isEnabled, functionId, recordInputs, recordOutputs }
- Framework delegates to Vercel AI SDK experimental_telemetry — spans created per model call + tool invocation
