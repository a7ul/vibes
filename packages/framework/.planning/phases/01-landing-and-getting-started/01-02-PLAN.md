**Part A: Create docs/getting-started/hello-world.mdx**

This is a single progressive tutorial using the `<Steps>` component. Structure:

1. **Frontmatter:**
   ```yaml
   ---
   title: "Hello World"
   description: "Build your first AI agent in 5 minutes. Start simple, add tools, structured output, and tests."
   ---
   ```

2. **Opening paragraph** (2-3 sentences): In this tutorial you will build a weather agent from scratch. You will start with a bare agent, add a tool, add structured output, and write a test -- all in one progressive flow. By the end, you will have a fully tested, type-safe agent.

3. **Prerequisites** (brief): Vibes installed (link to install page), an Anthropic API key set in env.

4. **Wrap entire tutorial in `<Steps>`:**

   **Step 1: Create a Bare Agent**
   - Title: "Create a bare agent"
   - Show the 8-line bare agent code (from research Step 1)
   - Brief explanation: Agent takes a model and system prompt. `agent.run()` sends a message and returns a result. `result.output` is the text response.
   - Add a `<Note>`: "Without tools, the agent can only respond from its training data. Let's fix that."

   **Step 2: Add a Tool**
   - Title: "Add a tool"
   - Show the full tool + agent code (from research Step 2)
   - Explain: `tool()` creates a type-safe tool with Zod parameter validation. The agent automatically calls tools when relevant. The `_ctx` parameter is the RunContext (we will use it later for dependency injection).
   - Add a `<Tip>`: "The tool's `description` helps the model decide when to call it. Be specific!"

   **Step 3: Add Structured Output**
   - Title: "Add structured output"
   - Show the outputSchema code (from research Step 3)
   - Explain: `outputSchema` forces the agent to return data matching your Zod schema. `result.output` is now typed as `{ city: string; temperature: number; condition: string; summary: string }` instead of `string`. The agent will retry if validation fails.
   - Add a `<Note>`: "Structured output uses tool-based extraction by default. The agent calls a special output tool internally."

   **Step 4: Test It**
   - Title: "Test your agent"
   - Show the test code (from research Step 4)
   - Explain: `TestModel` simulates model responses without API calls. `agent.override()` creates a copy of the agent with the test model (immutable -- original agent unchanged). `setAllowModelRequests(false)` blocks any accidental real API calls in your test suite.
   - Show Deno test run command: `deno test agent_test.ts`

5. **What's Next?** CardGroup with 2-3 cards linking to concepts pages (agents, tools, dependencies) -- use placeholder paths that will exist after Phase 2: `/concepts/agents`, `/concepts/tools`, `/concepts/dependencies`. Wrap in a `<Note>` saying "These concept deep-dives are coming soon."
