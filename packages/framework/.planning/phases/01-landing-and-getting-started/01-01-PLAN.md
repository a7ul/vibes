3. **Design Philosophy** section with 4-5 principles:
   - "Type safety everywhere" -- Zod schemas for tools, output, validation. No `any` types leaking.
   - "Dependency injection, not globals" -- RunContext carries deps through the entire call chain. Test by swapping deps, not mocking modules.
   - "Model-agnostic by default" -- Vercel AI SDK handles the model layer. Switch providers by changing one line.
   - "Test-first design" -- TestModel, FunctionModel, agent.override(), setAllowModelRequests(false). Every agent is testable without API calls.
   - "Progressive complexity" -- Start with 5 lines, add tools, add structured output, add DI, add streaming. Each layer is opt-in.

4. **Standing on the Shoulders of Giants** section:
