<!-- Verified APIs from 04-RESEARCH.md. These are the ONLY correct APIs. -->

A2AAdapter:
  constructor: new A2AAdapter(agent, options?)
  options: { name?, description?, url?, version?, skills?, provider?, deps?, taskStore? }
  handler(): (req: Request) => Promise<Response>
  handleRequest(req: Request): routes GET /.well-known/agent.json and POST /

JSON-RPC methods (all via POST /):
  message/send or tasks/send     → synchronous, returns full A2ATask
  message/stream or tasks/sendSubscribe → SSE streaming
  tasks/get                      → fetch task by ID
  tasks/cancel                   → cancel running task

A2ATaskState: submitted → working → completed | failed | canceled
  (also: input-required, not yet fully wired)

MemoryTaskStore:
  In-memory only, no persistence across restarts

TemporalAgent:
  constructor: new TemporalAgent(agent, { taskQueue, depsFactory?, modelCallActivity?, toolCallActivity? })
  activities   — PROPERTY (not method), shape: { runModelTurn, runToolCall }
  workflowFn   — PROPERTY (async function, valid Temporal workflow)
  run(prompt, opts?)   — non-Temporal fallback
  taskQueue getter

MockTemporalAgent:
  constructor: new MockTemporalAgent(agent, { taskQueue: "test" })  <!-- second arg REQUIRED -->
  run(prompt, opts?)
  simulateReplay(prompt, opts?)
  getActivityHistory(): ReadonlyArray<ActivityHistoryEntry>
  reset()

Serialization (lib/temporal/serialization.ts):
  serializeRunState(messages)     <!-- NOT serializeAgentState -->
  deserializeRunState(messages)   <!-- NOT deserializeAgentState -->
  roundTripMessages(messages)

Vercel AI UI pattern (no framework adapter class):
  agent.stream(prompt) → result.textStream → toDataStreamResponse() from "ai" package
  Frontend: useChat({ api: "/api/chat" }) from "ai/react"

Part A — Create vercel-ai-ui.mdx:

There is no dedicated Vibes framework class for this integration. The pattern is:
1. Vibes agent.stream() → result.textStream
2. toDataStreamResponse() from the "ai" package converts to Vercel AI data stream protocol
3. Frontend useChat/useCompletion hooks connect to the API route

Page structure:
1. Title: "Vercel AI UI" with description frontmatter
2. Brief: what Vercel AI UI is (useChat, useCompletion React hooks for streaming UIs), link to sdk.vercel.ai/docs/ai-sdk-ui
3. How streaming works end-to-end — architecture diagram (Mermaid sequenceDiagram):
   React useChat → POST /api/chat → API route → agent.stream() → toDataStreamResponse() → data stream → useChat renders tokens
4. API route section — Next.js App Router example:
   ```typescript
   import { agent } from "@/lib/agent";
   import { toDataStreamResponse } from "ai";

   export async function POST(req: Request) {
     const { messages } = await req.json();
     const lastMessage = messages[messages.length - 1];
     const result = await agent.stream(lastMessage.content, {
       messageHistory: messages.slice(0, -1),
     });
     return toDataStreamResponse(result.textStream);
   }
   ```
5. React frontend section — useChat hook:
   ```typescript
   import { useChat } from "ai/react";

   export function Chat() {
     const { messages, input, handleInputChange, handleSubmit } = useChat({
       api: "/api/chat",
     });
     // render messages and form
   }
   ```
6. useCompletion section — for single-turn completions (simpler, no history):
   ```typescript
   const { completion, input, handleInputChange, handleSubmit } = useCompletion({
     api: "/api/complete",
   });
   ```
7. Deno server example — for non-Next.js setups using Deno.serve:
   ```typescript
   Deno.serve(async (req) => {
     const { prompt } = await req.json();
     const result = await agent.stream(prompt);
     return toDataStreamResponse(result.textStream);
   });
   ```
8. Note: the "ai" package (v6) is already a dependency in deno.json — no separate install needed for Deno
9. Structured output note — if agent uses outputSchema, use result.partialOutput stream instead of textStream, and handle with useCompletion
10. API reference: key Vercel AI UI hooks and the toDataStreamResponse() function
