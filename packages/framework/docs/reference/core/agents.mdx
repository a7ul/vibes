An `Agent` is the central object in Vibes. It holds a model, system prompts, tools, an optional output schema, and result validators. You call `.run()` or `.stream()` on it to execute a prompt.

## Creating an Agent

```ts
import { Agent } from "@vibes/framework";
import { anthropic } from "@ai-sdk/anthropic";

const agent = new Agent({
  model: anthropic("claude-haiku-4-5-20251001"),
  systemPrompt: "You are a helpful assistant.",
});
```

## `AgentOptions`

| Option             | Type                                        | Default  | Description                                               |
| ------------------ | ------------------------------------------- | -------- | --------------------------------------------------------- |
| `model`            | `LanguageModel`                             | required | Any Vercel AI SDK model                                   |
| `name`             | `string`                                    | -        | Human-readable label for the agent                        |
| `systemPrompt`     | `string \| SystemPromptFn<TDeps>`           | -        | Static string or function called with `RunContext`        |
| `instructions`     | `string \| InstructionsFn<TDeps>`           | -        | Dynamic function appended after `systemPrompt`            |
| `tools`            | `ToolDefinition[]`                          | -        | Tools the model can call                                  |
| `toolsets`         | `Toolset[]`                                 | -        | Toolset instances (resolved per-turn)                     |
| `outputSchema`     | `ZodType \| ZodType[]`                      | -        | Zod schema for structured output                          |
| `outputMode`       | `"tool" \| "native" \| "prompted"`          | `"tool"` | How structured output is requested                        |
| `resultValidators` | `ResultValidator[]`                         | -        | Validators run after output is parsed                     |
| `historyProcessors`| `HistoryProcessor[]`                        | -        | Transform message history before each model call          |
| `maxRetries`       | `number`                                    | `3`      | Max retries for validation failures                       |
| `maxTurns`         | `number`                                    | `10`     | Max tool-call round trips per run                         |
| `endStrategy`      | `"early" \| "exhaustive"`                   | `"early"`| When to stop after a tool-only turn                       |
| `maxConcurrency`   | `number`                                    | -        | Max tools executing in parallel per turn                  |
| `usageLimits`      | `UsageLimits`                               | -        | Token/request budget guard                                |
| `modelSettings`    | `ModelSettings`                             | -        | temperature, maxTokens, topP, etc.                        |
| `telemetry`        | `TelemetrySettings`                         | -        | OpenTelemetry tracing configuration                       |

## Type Parameters

```ts
class Agent<TDeps = undefined, TOutput = string>
```

- **`TDeps`** - the type of dependencies injected at run time via `deps`. Defaults to `undefined` (no deps).
- **`TOutput`** - the type of `result.output`. Defaults to `string`. Set by providing an `outputSchema`.

```ts
type Deps = { db: Database };
const agent = new Agent<Deps, MyOutput>({
  model: ...,
  outputSchema: MyOutputSchema,
});
```

## Running an Agent

### `.run(prompt, opts?)`

Executes the agent and waits for the final result.

```ts
const result = await agent.run("What is the capital of France?");
console.log(result.output);   // "Paris"
console.log(result.usage);    // { inputTokens, outputTokens, totalTokens, requests }
console.log(result.messages); // full ModelMessage[] history
console.log(result.runId);    // unique run identifier
```

**Options:**

| Option           | Type              | Description                                                  |
| ---------------- | ----------------- | ------------------------------------------------------------ |
| `deps`           | `TDeps`           | Runtime dependencies passed to tools and dynamic prompts     |
| `messageHistory` | `ModelMessage[]`  | Prior messages to prepend (enables multi-turn conversations) |
| `usageLimits`    | `UsageLimits`     | Per-run budget override                                      |
| `modelSettings`  | `ModelSettings`   | Per-run model settings override                              |

Returns [`RunResult<TOutput>`](./run-context#runresult).

### `.stream(prompt, opts?)`

Starts the agent and returns a [`StreamResult`](./streaming) immediately. Text deltas are available as an async iterable.

```ts
const stream = agent.stream("Tell me a story.");

for await (const chunk of stream.textStream) {
  process.stdout.write(chunk);
}

const output = await stream.output;
```

Same options as `.run()`. See [Streaming](./streaming) for full details.

## Dynamic System Prompts

Dynamic system prompts are the equivalent of pydantic-ai's `@agent.system_prompt` decorator - one of the most powerful patterns in the framework.

`systemPrompt` can be a function that receives a `RunContext` at runtime:

```ts
type Deps = { username: string; role: string };

const agent = new Agent<Deps>({
  model,
  systemPrompt: (ctx) =>
    `You are an assistant for ${ctx.deps.username}. Their role is: ${ctx.deps.role}.`,
});
```

### `addSystemPrompt()`

Add system prompts after construction - the equivalent of pydantic-ai's `@agent.system_prompt` decorator:

```ts
const agent = new Agent({
  model,
  systemPrompt: "You are a helpful assistant.",
});

// Static addition
agent.addSystemPrompt("Always respond in bullet points.");

// Dynamic - called every run with the current RunContext
agent.addSystemPrompt((ctx) => `Today is ${new Date().toDateString()}.`);

// With deps
agent.addSystemPrompt((ctx) => `Logged-in user: ${ctx.deps.username}`);
```

All prompts (construction + added) are joined with `\n\n` into a single system message.

> **pydantic-ai mapping:** `agent.addSystemPrompt(fn)` is the equivalent of:
> ```python
> @agent.system_prompt
> def inject_user(ctx: RunContext[Deps]) -> str:
>     return f"Logged-in user: {ctx.deps.username}"
> ```

### `instructions` vs `systemPrompt`

Both configure the agent's system instructions and can be strings or functions:

- `systemPrompt` - resolved first
- `instructions` - resolved after `systemPrompt`, appended to it

```ts
const agent = new Agent<{ locale: string }>({
  model,
  systemPrompt: "You are a helpful assistant.",
  instructions: (ctx) => `Always respond in ${ctx.deps.locale}.`,
});
```

Use `systemPrompt` for stable base configuration and `instructions` for per-run dynamic context.

## Adding Tools and Validators After Construction

```ts
agent.addTool(myTool);
agent.addToolset(myToolset);
agent.addSystemPrompt("Always respond in bullet points.");
agent.addInstruction((ctx) => `User: ${ctx.deps.username}`);
agent.addResultValidator(myValidator);
agent.addHistoryProcessor(trimHistoryProcessor(20));
```

## Model Selection

Any model implementing `LanguageModel` from the Vercel AI SDK works:

```ts
import { anthropic } from "@ai-sdk/anthropic";
import { openai } from "@ai-sdk/openai";

const agent = new Agent({ model: anthropic("claude-sonnet-4-6") });
const agent = new Agent({ model: openai("gpt-4o") });
```

## Multi-Turn Conversations

Pass `result.messages` from one run as `messageHistory` in the next:

```ts
const first = await agent.run("My name is Alice.");
const second = await agent.run("What is my name?", {
  messageHistory: first.messages,
});
console.log(second.output); // "Your name is Alice."
```

## Turn Limit

Each call to `.run()` or `.stream()` runs an internal loop. If `maxTurns` is reached, a [`MaxTurnsError`](./errors) is thrown.

```ts
const agent = new Agent({ model: ..., maxTurns: 5 });
```

## `endStrategy`

Controls when the agent considers a run complete after a tool-only turn:

| Value               | Behaviour                                                                                                              |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `"early"` (default) | Stop as soon as the model produces a text response or output tool call, even if other tools were also called this turn |
| `"exhaustive"`      | Run all tool calls in the current turn before checking for completion                                                  |

```ts
const agent = new Agent({ model, endStrategy: "exhaustive" });
```

Use `"exhaustive"` when your tools have side effects that all need to complete before the model summarises.

## `maxConcurrency`

Limits how many tools execute in parallel within a single turn. Defaults to unlimited.

```ts
const agent = new Agent({
  model,
  tools: [searchTool, fetchTool, parseTool],
  maxConcurrency: 2,
});
```

## `runStreamEvents()`

For detailed event-by-event observation of a run, use `runStreamEvents()`. Each event has a `kind` discriminant:

```ts
for await (const event of agent.runStreamEvents("Tell me a joke.")) {
  if (event.kind === "text-delta") process.stdout.write(event.delta);
  if (event.kind === "tool-call-start") console.log("called", event.toolName);
  if (event.kind === "final-result") console.log("done", event.output);
}
```

See [Streaming](./streaming#runstreamevents) for the full event type reference.
