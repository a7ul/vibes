This document tracks every pydantic-ai feature and its status in the TypeScript
framework. Use it as a backlog when deciding what to port next.

**Legend:** ✅ Ported · 🚧 Partial · ❌ Not ported

## Agent API

| Feature                 | pydantic-ai                                  | Status | Docs | Notes |
| ----------------------- | -------------------------------------------- | ------ | ---- | ----- |
| `agent.run()`           | `agent.run(prompt, deps=x)`                  | ✅     | [Agents](/reference/core/agents) | `agent.run(prompt, { deps: x })` |
| `agent.run_stream()`    | `agent.run_stream(prompt)`                   | ✅     | [Streaming](/reference/core/streaming) | `agent.stream(prompt)` |
| Agent name              | `agent.name`                                 | ✅     | [Agents](/reference/core/agents) | `name` on `AgentOptions` |
| System prompt (static)  | `system_prompt="..."`                        | ✅     | [Agents](/reference/core/agents) | `systemPrompt: "..."` |
| System prompt (dynamic) | `@agent.system_prompt` decorator             | ✅     | [Agents](/reference/core/agents) | `agent.addSystemPrompt(fn)` or `systemPrompt: [fn]` |
| Tools                   | `@agent.tool` / `tools=[...]`                | ✅     | [Tools](/reference/core/tools) | `agent.addTool(tool({...}))` |
| Structured output       | `result_type: BaseModel`                     | ✅     | [Structured Output](/reference/core/structured-output) | `outputSchema: z.object({...})` |
| Result validators       | `@agent.result_validator`                    | ✅     | [Result Validators](/reference/core/result-validators) | `agent.addResultValidator(fn)` |
| Max retries             | `max_retries` / `max_result_retries`         | ✅     | [Agents](/reference/core/agents) | `maxRetries` on `AgentOptions` |
| Max turns               | `max_turns`                                  | ✅     | [Agents](/reference/core/agents) | `maxTurns` on `AgentOptions` |
| Message history         | `message_history=`                           | ✅     | [Message History](/reference/advanced/message-history) | `{ messageHistory: [...] }` on `run()` |
| Metadata tagging        | `metadata=` on run                           | ✅     | [Agents](/reference/core/agents) | `{ metadata: {...} }` on `run()`/`stream()` - accessible via `ctx.metadata` |
| `Agent.override()`      | Context manager swapping model/deps/toolsets | ✅     | [Testing](/reference/core/testing) | `agent.override({ model, tools, ... }).run(prompt)` |
| Event-stream run        | `agent.run_stream_events()`                  | ✅     | [Streaming](/reference/core/streaming) | `agent.runStreamEvents(prompt)` - async iterable of typed `AgentStreamEvent` objects |
| End strategy            | `end_strategy`                               | ✅     | [Agents](/reference/core/agents) | `endStrategy: 'early' \| 'exhaustive'` on `AgentOptions`/`RunOptions` |
| Max concurrency         | `max_concurrency`                            | ✅     | [Agents](/reference/core/agents) | `maxConcurrency` on `AgentOptions` - semaphore-based cap on concurrent tool executions |
| `instructions` field    | `@agent.instructions` decorator              | ✅     | [Agents](/reference/core/agents) | `instructions` on `AgentOptions`/`RunOptions`; re-injected each turn, not stored in message history |
| Model-specific settings | `model_settings=` on `run()`                 | ✅     | [Model Settings](/reference/advanced/model-settings) | `modelSettings: { temperature, maxTokens, ... }` on `AgentOptions` or `RunOptions` |
| Sync run                | `agent.run_sync()`                           | ❌     | — | Deno is async-native - not applicable |
| Node-level iteration    | `agent.iter()` / `AgentRun`                  | ❌     | — | Not applicable - use `runStreamEvents()` for step-by-step observation in async TypeScript |
| Last run messages       | `agent.last_run_messages`                    | ❌     | — | Removed from pydantic-ai; superseded by `result.newMessages` |

## Tools

| Feature                | pydantic-ai                                     | Status | Docs | Notes |
| ---------------------- | ----------------------------------------------- | ------ | ---- | ----- |
| Tools with context     | `@agent.tool`                                   | ✅     | [Tools](/reference/core/tools) | `tool({ execute: (ctx, args) => ... })` |
| Tool `maxRetries`      | `retries=` on `@agent.tool`                     | ✅     | [Tools](/reference/core/tools) | `maxRetries` on `ToolDefinition` |
| Plain tools (no ctx)   | `@agent.tool_plain`                             | ✅     | [Tools](/reference/core/tools) | `plainTool({ name, description, parameters, execute })` |
| Tool `prepare` method  | `prepare=` on `Tool` class                      | ✅     | [Tools](/reference/core/tools) | `prepare: (ctx) => tool \| null` on `ToolDefinition` |
| `args_validator`       | `args_validator=` on tool                       | ✅     | [Tools](/reference/core/tools) | `argsValidator: (args) => void` on `ToolDefinition` |
| `Tool.from_schema()`   | Build tool from raw JSON schema                 | ✅     | [Tools](/reference/core/tools) | `fromSchema({ name, description, jsonSchema, execute })` |
| Multi-modal returns    | Return images / audio / binary from tools       | ✅     | [Multi-Modal](/reference/advanced/multi-modal) | `BinaryContent` / `BinaryImage` - returned from `execute`, auto-converted |
| `UploadedFile` support | `UploadedFile` for provider file uploads        | ✅     | [Multi-Modal](/reference/advanced/multi-modal) | `UploadedFile` type + `uploadedFileSchema` for tool parameters |
| Tool result metadata   | Attach metadata keyed by `tool_call_id`         | ✅     | [Tools](/reference/core/tools) | `ctx.attachMetadata(toolCallId, meta)` - exposed on `result.toolMetadata` |
| Output functions       | Final-action tools (no model feedback loop)     | ✅     | [Structured Output](/reference/core/structured-output) | `outputTool({ ... })` - sets `isOutput: true`, ends run on call |
| Sequential execution   | `sequential=True` on tool                       | ✅     | [Tools](/reference/core/tools) | `sequential: true` on `ToolDefinition` - acquires mutex before executing |
| Deferred tools         | Tools requiring human approval before execution | ✅     | [Deferred Tools](/reference/advanced/deferred-tools) | `requiresApproval: true` on `ToolDefinition` - see Deferred Tools section |
| MCP server tools       | Connect external MCP servers as tool providers  | ✅     | [MCP](/reference/integrations/mcp) | `MCPToolset` wraps any `MCPClient` - see MCP section |
| Docstring extraction   | Auto-doc from Python docstrings                 | ❌     | — | No runtime equivalent in TypeScript - use `description` field explicitly |

## Deferred Tools (Human-in-the-Loop & External Execution)

| Feature                      | pydantic-ai                                      | Status | Docs | Notes |
| ---------------------------- | ------------------------------------------------ | ------ | ---- | ----- |
| `requires_approval=True`     | Mark a tool as approval-required                 | ✅     | [Deferred Tools](/reference/advanced/deferred-tools) | `requiresApproval: true` on `ToolDefinition` or `tool()` options |
| `ApprovalRequired` exception | Pause agent, surface pending calls to caller     | ✅     | [Human in the Loop](/guides/human-in-the-loop) | `ApprovalRequiredError` - catch it, inspect `.requests`, resume with results |

| `DeferredToolResults`        | Provide approved (or overridden) results         | ✅     | [Human in the Loop](/guides/human-in-the-loop) | `agent.resume(deferred, { results: [...] })` or `run(..., { deferredResults })` |
| Argument override on resume  | Modify args during approval before execution     | ✅     | [Deferred Tools](/reference/advanced/deferred-tools) | `argsOverride` field on `DeferredToolResult` |
| `CallDeferred` exception     | Defer a tool call to an external process         | ✅     | [Deferred Tools](/reference/advanced/deferred-tools) | `ExternalToolset` raises `ApprovalRequiredError` for all tools |
| `ExternalToolset`            | Accept raw JSON schema tools for deferred calls  | ✅     | [Toolsets](/reference/core/toolsets) | `new ExternalToolset([{ name, description, jsonSchema }])` |

## Output & Structured Results

| Feature                     | pydantic-ai                          | Status | Docs | Notes |
| --------------------------- | ------------------------------------ | ------ | ---- | ----- |
| Single schema output        | `result_type: BaseModel`             | ✅     | [Structured Output](/reference/core/structured-output) | `outputSchema: z.object({...})` via `final_result` tool |
| Result validators           | `@agent.result_validator`            | ✅     | [Result Validators](/reference/core/result-validators) | `addResultValidator(fn)` - throw to retry |
| `result.all_messages()`     | Full message history                 | ✅     | [Message History](/reference/advanced/message-history) | `result.messages` (full) + `result.newMessages` (this run) |
| `result.new_messages()`     | Messages added in _this_ run only    | ✅     | [Message History](/reference/advanced/message-history) | `result.newMessages` on `RunResult` and `StreamResult` |
| `@agent.output_validator`   | Validate output post-parse           | ✅     | [Result Validators](/reference/core/result-validators) | Covered by `addResultValidator` |
| Union output types          | `output_type=[TypeA, TypeB]`         | ✅     | [Structured Output](/reference/core/structured-output) | `outputSchema: [schemaA, schemaB]` - registers `final_result_0`, `_1`… |
| Native structured output    | `NativeOutput` marker class          | ✅     | [Structured Output](/reference/core/structured-output) | `outputMode: 'native'` - uses AI SDK `Output.object()` / JSON mode |
| Prompted output mode        | `PromptedOutput` marker class        | ✅     | [Structured Output](/reference/core/structured-output) | `outputMode: 'prompted'` - schema injected into system prompt |
| Streaming structured output | Partial validation as output streams | ✅     | [Streaming](/reference/core/streaming) | `result.partialOutput` async iterable on `StreamResult` |
| Message serialization       | `ModelMessagesTypeAdapter`           | ✅     | [Message History](/reference/advanced/message-history) | `serializeMessages(msgs)` / `deserializeMessages(json)` |
| Disable schema prompt       | `template=False` on output marker    | ✅     | [Structured Output](/reference/core/structured-output) | `outputTemplate: false` on `AgentOptions` |
| `BinaryImage` output        | Generate images as output type       | ✅     | [Multi-Modal](/reference/advanced/multi-modal) | `outputSchema: BINARY_IMAGE_OUTPUT` - first tool result with `image/*` MIME type becomes the run output as `BinaryContent` |
