## v0.4.0 — 2025-06-10

### Added

- **A2A adapter** (`A2AAdapter`) — expose any Vibes agent as an Agent-to-Agent (A2A) protocol server
- **AG-UI adapter** (`AGUIAdapter`) — stream Vibes agent events to AG-UI compliant frontends
- **Temporal agent** (`TemporalAgent`) — durable, long-running agent workflows backed by Temporal
- Multi-modal content helpers: `imageMessage`, `audioMessage`, `fileMessage` for attaching binary content to agent runs
- `MCPToolset` and `MCPManager` for loading tools from Model Context Protocol servers

### Changed

- `agent.stream()` now returns an async iterable of typed `AgentStreamEvent` objects; use `event.kind` (not `event.type`) to discriminate events
- `runStreamEvents()` added as a lower-level alternative to `agent.stream()` for custom event handling

## v0.3.0 — 2025-03-20

### Added

- **Graph workflows** (`Graph`, `BaseNode`) — directed-graph agent orchestration with typed edges
- **Human-in-the-loop** (`requiresApproval`) — deferred tools that pause execution and resume with a human-provided value via `agent.resume()`
- **OTel instrumentation** (`instrumentAgent`) — wrap any agent with OpenTelemetry tracing; spans follow Vercel AI SDK naming conventions
- `TestModel` and `FunctionModel` improvements: `TestModel` now supports `textResponse`, `toolCalls`, and `callsTools` options

### Changed

- `agent.override()` now accepts partial options; previously required all fields

### Fixed

- `RunContext.attachMetadata()` now correctly merges metadata across nested tool calls

---

## v0.2.0 — 2024-12-05

### Added

- **Streaming** (`agent.stream()`) — stream partial text responses and tool call events
- **Result validators** (`resultValidators`) — validate and optionally retry structured output
- **Agent-as-tool** — register a Vibes agent as a tool on another agent for multi-agent workflows
- `setAllowModelRequests(false)` — globally disable real model requests in test environments
- `plainTool()` — register a tool that receives the full `RunContext` without a Zod input schema
- `outputTool()` — register a structured-output tool that terminates the agent run
- `fromSchema()` — create a tool from an existing JSON Schema object

### Fixed

- Type inference for `outputSchema` when using deeply nested Zod objects

---

## v0.1.0 — 2024-09-15

### Added

- **`Agent` class** — core agent with `agent.run()`, system prompt, model configuration
- **`RunContext`** — type-safe dependency injection context passed to every tool and validator call; fields: `deps`, `messages`, `usage`, `modelOutput`, `runId`, `abortSignal`, `outputFormat`; method: `attachMetadata()`
- **`tool()`** — define typed tools with Zod input schemas
- **Structured output** (`outputSchema`) — Zod-validated structured responses with automatic retry on schema mismatch
- **`TestModel`** and **`FunctionModel`** — test utilities for unit testing agents without real API calls
- **`agent.override()`** — override model, deps, or tools for a single run (useful in tests)
- Vercel AI SDK provider compatibility: `@ai-sdk/anthropic`, `@ai-sdk/openai`, `@ai-sdk/google`, and all other AI SDK providers work out of the box
