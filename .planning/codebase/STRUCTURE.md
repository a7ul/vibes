## 3. Naming Conventions

| Pattern | Example | Convention |
|---------|---------|-----------|
| Files | `mcp_toolset.ts` | `snake_case.ts` |
| Classes | `Agent`, `Graph`, `DeferredToolRequests` | `PascalCase` |
| Interfaces | `Toolset`, `ToolDefinition`, `RunContext` | `PascalCase` |
| Type aliases | `HistoryProcessor`, `EndStrategy` | `PascalCase` |
| Functions | `executeRun`, `createRunContext` | `camelCase` |
| Test files | `agent_test.ts` | `{module}_test.ts` |
| Internal files | `_run_utils.ts`, `_helpers.ts` | `_snake_case.ts` |
| Barrel files | `mod.ts` | Always named `mod.ts` |

## 5. Public API (`mod.ts` exports)

The framework exports:
- `Agent` class + `AgentOptions` type
- `tool()` factory + `ToolDefinition` type
- All toolset classes
- History processors
- MCP integration (`MCPToolset`, config types)
- Graph (`Graph`, `BaseNode`, `StatePersistence`)
- Temporal (`TemporalAgent`)
- Multimodal types
- OTel settings
- Testing utilities (`TestModel`, `FunctionModel`)
- AG-UI adapter
- All error types + result types
