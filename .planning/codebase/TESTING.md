## 3. Test Patterns

### Basic Agent Test Pattern
```typescript
import { assertEquals, assertExists } from "@std/assert";
import { Agent, tool } from "../mod.ts";
import { textResponse, MockLanguageModelV3 } from "./_helpers.ts";
import { z } from "zod";

Deno.test("Agent - basic text run", async () => {
  const model = new MockLanguageModelV3({
    doGenerate: textResponse("hello world"),
  });
  const agent = new Agent({ model, systemPrompt: "Be concise." });
  const result = await agent.run("Say exactly: hello world");
  assertEquals(result.output, "hello world");
  assertExists(result.runId);
});
```

### Tool Call Test Pattern
```typescript
Deno.test("Agent - tool execution", async () => {
  const model = new MockLanguageModelV3({
    doGenerate: mockValues(
      toolCallResponse("my_tool", { arg: "value" }),
      textResponse("done"),
    ),
  });
  const myTool = tool({
    name: "my_tool",
    description: "...",
    parameters: z.object({ arg: z.string() }),
    execute: async (ctx, args) => args.arg,
  });
  const agent = new Agent({ model, tools: [myTool] });
  const result = await agent.run("Use the tool");
  assertEquals(result.output, "done");
});
```

### Structured Output Test Pattern
```typescript
Deno.test("Agent - structured output", async () => {
  const OutputSchema = z.object({ name: z.string(), capital: z.string() });
  const model = new MockLanguageModelV3({
    doGenerate: toolCallResponse("final_result", { name: "France", capital: "Paris" }),
  });
  const agent = new Agent({ model, outputSchema: OutputSchema });
  const result = await agent.run("...");
  assertEquals(result.output.name, "France");
});
```

## 4. Mocking Strategy

### MockLanguageModelV3 (from `ai/test`)
- Direct mock of the Vercel AI SDK model interface
- `doGenerate` function controls responses turn-by-turn

### Helper Functions (`tests/_helpers.ts`)
- `textResponse(text)` — returns a model response with plain text
- `toolCallResponse(name, args)` — returns a model response calling a tool
- `mockValues(...responses)` — cycles through multiple responses (one per turn)

### TestModel (`lib/testing/test_model.ts`)
- Higher-level mock: automatically generates valid tool calls based on schemas
- Turn 1: calls all non-`final_result` tools with schema-valid args
- Turn 2: calls `final_result` with schema-valid data (or returns text)
- Use when you want realistic auto-behavior without specifying every turn

### FunctionModel (`lib/testing/function_model.ts`)
- Most flexible: provide a custom `doGenerate` function
- Full control over every aspect of the model response

## 5. Test Organization

- **One test file per module**: `mcp_toolset_test.ts` tests `lib/mcp/mcp_toolset.ts`
- **No test subdirectories**: All tests flat in `tests/`
- **`_helpers.ts`** prefix convention for shared utilities (not a test file itself)
- Tests are named descriptively: `"Agent - basic text run"`, `"MCPToolset - tool discovery"`

## 8. Test Anti-Patterns to Avoid

- Do NOT use real API keys or make real model calls in tests (use mocks)
- Do NOT share mutable state between `Deno.test()` blocks
- Do NOT test implementation details — test observable behavior
- DO assert on specific values, not just "truthy"
- DO test error paths (`assertRejects`) not just happy paths
